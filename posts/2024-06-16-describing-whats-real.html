The Real Numbers and a Theory of Knowledge

Posted on June 16, 2024 by Travis Athougies
in
  • math
  • philosophy

In the previous post, we talked about the various ‘levels’ of infinity and how they come about through relatively simple ideas.

To recap, we went over the natural numbers as the first example of an infinite of things.

We then considered how to make addition and multiplication nice when dealing with equations of the form \(x + y = z\) or \(x \times y = z\). That is we wanted to be able to solve for any one of \(x\), \(y\), or \(z\) given the other two. We added the negative numbers and reciprocals to make this possible.

But, then we decided to look at equations of the form \(x + x = y\) or \(x \times x = y\). We find that for \(x + x = y\), we can solve for \(x\) given \(y\) by just doing \(x = \frac{y}{2}\). However, for \(x \times x = y\), we found that there are choices for \(y\) that make the equation impossible to solve unless we add yet more numbers. But these numbers were fundamentally different, and we found that there are a lot more of these numbers then there are of the normal counting numbers, despite both sets being infinite.

In this post, I want to consider how it is I was able to describe these numbers which transcend infinity in a blog. That is – how are we able to describe infinite things, even things more infinite than the numbers – in a finite space.

To start with I want to observe the following. Consider \(\frac{1}{3}\). Most of us learn in grade school that this is equivalent to

\[0.\overline{3333}…\]

where the \(\overline{333}\) indicates that this pattern repeats indefinitely. This is an infinite decimal expansion of \(\frac{1}{3}\), but it’s only infinite when written as a decimal. There are other representations of \(\frac{1}{3}\) that are not infinite. For example, \(\frac{1}{3}\) is a simple finite representation. Alternatively, we can write \(\frac{1}{3}\) in base 3 as \(0.1\). Moreover, the line over the 3s indicating that the go on forever actually uniquely identifies \(\frac{1}{3}\). No matter what we pick, this is a number that is solidly identified in a finite space.

Either way, just because there are some representations of \(\frac{1}{3}\) that require an infinite representation, there do exist others that do not.

But not all numbers are so nice. The number we looked at in the last post was \(\sqrt{2}\), which is the number that yields two when multiplied by itself. The decimal expansion of \(\sqrt{2}\) looks like

\[1.41415962…\]

Note here that there is no line over any digits. There is no repetition. That is to say that this expansion goes on forever, but it never ends in a pattern that repeats indefinitely. There might be large swaths of numbers in the expansion where digits repeat over and over again, but eventually, these will end and some other new sequence will begin.

Even worse, unlike \(\frac{1}{3}\). We cannot write \(\sqrt{2}\) as a fraction, because it’s not rational. But we can give you ever closer approximations. In the last post, we came up with a general procedure to do this, and get ever closer.

So another way to represent the above is the following:

\[ \{ \frac{3}{2}, \frac{17}{12}, \frac{577}{408}, \ldots \} \]

This is an application of Newton’s method for finding roots (discussed in the last post). We showed that this too will go on forever.

So what representation can we use for this number? The obvious answer of course is to just use \(\sqrt(2)\). That’s compact and describes exactly what we want. But what if we want to add \(\sqrt(2)\) to itself. In the rational case of \(\frac{1}{3}\), adding \(\frac{1}{3}\) to itself yields a similar finite representation of the repeating decimal: \(\frac{2}{3}\). But if I add \(\sqrt{2}\) to itself, then I just get \(2\sqrt{2}\), which is a finite representation, but not of the same form as \(\sqrt{2}.\) Moreover, if I start adding \(\sqrt{3} + \sqrt{2}\), I can’t reduce them either.

What we want to do now is to come up with some canonical way to describe the real numbers.

In the last post, we saw that the real numbers arose when we tried to find the ‘least’ element of infinite sets of the rational numbers (or equivalently, the largest). In the case of the fraction sequence for \(\sqrt{2}\), these numbers get ever closer together, but there is no least value. At least, they had no least value that was rational. But, we just said that we could name these least values the reals.

So then this is what we’ll use to describe the reals. We’ll say they’re the least values of any set of the rational numbers.

Sets that are finite have an obvious representation: we just write out all the values.

Sets that are infinite do not have an obvious representation. We need some way to write out infinite sets, but we can’t do that in general. Can we?

We have to ask what it means to be an infinite collection of things. How would we even talk about this? Ony way is to suppose I had a magic elf that just tells me, given some number, wheter it’s in that set. Then, to represent this set finitely, I’d just point to the elf as my finite representative.

Unfortunately, we have no magic elves. Just our five senses and our minds.

What we need really is a way to describe this elf’s behavior. In other words, if I can describe what the elf would do in every situation, then I would have a way to represent my real number.

One thing about sets is that if an element is in a set, then it always is. That is, the elf will always give the same answer given the same input. This means that the elf is following a deterministic process to answer my question. [Alan Turing] tells us that anything following such a deterministic process can (theoretically) be described as a computer program. In other words, as long as the elf is being deterministic, we theoretically ought to be able to write a finite computer program that will return the same answers.

So, then this is our way, in general of determining whether a rational is in a set. I will simply give you a program that determines if it is. This program can be written in Python, C++, or a blog post. It doesn’t matter. If there’s a deterministic set of rules by which we can say a number is in the set, then I can describe those rules using any syntax in any language – natural or contrived.

So, for example, for the square root of two above, I can describe to you how I came up with those fractions (it’s described in my previous post), and now you have a description of a set with the proprties we wanted. Given a rational number, you can tell me if it’s part of the sequence of numbers or not. You can either do the math by hand, or write an automated program that does it for you (or come up with a design in the Game of Life, or something else).

But there’s a problem… do you see it?

Whatever I describe to you is made of words. Words are made of letters. There are 26 letters.

If you consider the set of all possible things I could say to you, it’s infinite, but it’s also countable, which means it’s smaller than the size of the real numbers.

Thus, we are forced to conclude that any finite representation of the real numbers will be unable to describe every possible real number. There is no way a countable number of things can describe every subset of the rational numbers, which is what a real number is.

Take a minute to ponder what we’ve done here. We started with strings of \(S\)s and \(Z\)s, and we’ve slowly extended it to a system that defies any finite explanation. In fact, some real numbers are so complicated that the only way to ‘write’ them is to write the digits down. These digits may be infinite, and there’s no deterministic way of saying which digit comes next. How can you write such a thing?

A note on description

Above, I purposefully avoided using the word ‘compute.’ Given the ubiquity of computers today, this word is loaded because it biases us to thinking of silicon-based machines. These are a kind of computer, sure, but computing is a much more broad idea. Turing showed that any number that can be described systematically by a human is equivalent to any number that can be calculated via his Turing machines. This makes quite a few systems Turing complete, meaning they’re able to calculate any of these numbers. These systems are automated – i.e., require no human intervention, and are deterministic in their operation. Examples include turing machines of course, but also the lambda calculus, the game of life, and even simpler systems, like Rule 110, which encodes in one number all rules required to compute any number that we can.

To quickly handle an objection, one can claim that they can describe a number that Turing machines cannot compute by defining a number equivalent to whether or not a program halts. For example, let

\[N = \prod{n = 0}_{\infty} P(n) \]

where \(P(n)\) is the probability that a Python program of length \(n\) halts. This number is seemingly definable and describable by humans, but not Turing machines.

There’s a logical flaw here though which is that the formula above is not a way to define the number. It presumes the existence of a function \(P\) that already exists. Indeed, if we allow ourselves to presume that \(P\) is given to us, we can write a computer program to compute that number:

def compute_N(p):
  r = 1
  n = 0
  while True:
    r *= P(n)
    n += 1

    yield r

So again, there is no contradiction. The function above exists, but since we cannot provide a proper \(p\) is unable to be computed. Nevertheless, it is definable in the same way we were able to do in natural language.

The set of all these numbers is called the definable reals.

Describing what’s real

So in our search for a canonical representation of the reals, we found that there are reals that are impossible to ever write down, canonically or not (in fact, most reals are this way). This is not like our inability to write down \(\sqrt{2}\) or \(\pi\) in decimal. There exist very good algorithms for finding digits of these numbers, even starting billions of digits in.

No these numbers are a wholly different thing. There are real numbers that can never be written down no matter what description we choose. They defy all finite description. The digits of these numbers are, at some point, not determinable. Being non-deterministic, they are – for lack of better word – random.

Describing what’s not real

If some super power from outside this universe were to choose such a number and run the universe according to its decimal (or binary) expansion. We’d have no way of possibly knowing whether the behavior was random or very fixed. It’s a logical impossibility.

Some interesting things:

  1. Initially, the idea that there are numbers that cannot be described really bothered mathematicians. Some mathematicians decided to ignore them completely and pretend they don’t exist. They only included reals which could be written down using deterministic algorithms. These are called the computable reals. It turns out large portions of mathematics can be re-stated in this way, but if we only include the computable reals, several properties of the real numbers change. For example, every function on the reals becomes continuous.
  2. If you haven’t noticed, these results mean it’s impossible to decide, in general, if two real numbers are equal, or less than, or greater than.
  3. The computable reals are closed under addition, multiplication, subtraction, division, etc. However, arbitrary subsets of the computable reals may or may not have a computable upper or lower limit. Think back to the never ending sequence for \(\sqrt{2}\). There are sets like this in the computable reals.

Conclusion

In a previous post, I discussed the fact that the game of life is intractable. In general, determining if it will arrive at particular pattern is not something anyone can ever do, period. It may have seemed that this idea of termination of a computer program and Cantor’s argument about diagonalizing the real numbers was different. In fact, it is the same argument.

In the next post, we’ll think about what this means for a topic we touched upon briefly but is fundamental to our understanding of causation, consciousness, and reality – randomness. We’ll talk about what it means for something to be random, and why this is linked to the nature of Turing machines / universal computation devices.
